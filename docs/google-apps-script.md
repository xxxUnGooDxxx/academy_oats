# Сбор результатов экзамена в Google Таблицу

Сайт после сдачи экзамена отправляет результат (имя, %, баллы, дата, сводка по блокам) в вашу Google Таблицу. Сервер не нужен — всё работает через бесплатный Google Apps Script.

## Шаг 1. Создать таблицу

1. Откройте [Google Таблицы](https://sheets.google.com) → создайте новую таблицу.
2. Переименуйте первый лист в **`Результаты`** (необязательно, но аккуратнее).
3. Название таблицы — например, «Академия ОАТС — результаты».

## Шаг 2. Добавить скрипт

1. В таблице: меню **Расширения → Apps Script**.
2. Удалите весь код в редакторе и вставьте содержимое файла **`google-apps-script.gs`** (лежит рядом с этой инструкцией).
3. Нажмите **Сохранить** (значок дискеты).

## Шаг 3. Опубликовать веб-приложение

1. В редакторе Apps Script: **Развернуть → Новое развёртывание**.
2. Тип (шестерёнка) → **Веб-приложение**.
3. Настройки:
   - **Описание**: любое (например, «приём результатов»).
   - **Запуск от имени**: *Я* (ваш аккаунт).
   - **У кого есть доступ**: **Все** (Anyone). Это важно — иначе сайт не сможет отправить данные.
4. Нажмите **Развернуть**. Google попросит **разрешить доступ** — подтвердите своим аккаунтом (на экране «приложение не проверено» → «Дополнительно» → «Перейти…» → «Разрешить»).
5. Скопируйте **URL веб-приложения** — он вида:
   `https://script.google.com/macros/s/XXXXXXXX/exec`

## Шаг 4. Вставить URL в сайт

Откройте файл `src/config.js` и вставьте URL:

```js
export const SHEETS_ENDPOINT =
  import.meta.env.VITE_SHEETS_ENDPOINT || 'https://script.google.com/macros/s/XXXXXXXX/exec'
```

Либо (надёжнее, чтобы URL не попал в git) создайте файл `.env` в корне проекта:

```
VITE_SHEETS_ENDPOINT=https://script.google.com/macros/s/XXXXXXXX/exec
```

Пересоберите/перезапустите проект (`npm run dev` или `npm run build`).

## Готово

После сдачи экзамена строка с результатом автоматически появится в таблице:

| Дата | Имя | Результат, % | Баллы | Из | Сдан | Блоков пройдено | Детали по блокам | Устройство |

## Частые вопросы

- **Ничего не приходит.** Проверьте, что в доступе стоит «Все», и что URL заканчивается на `/exec` (а не `/dev`).
- **Поменяли код скрипта — не работает.** После правки скрипта нужно сделать **новое развёртывание** (или «Управление развёртываниями → Изменить → Новая версия»).
- **Безопасность.** Отправка идёт «вслепую» (no-cors): сайт не читает ответ. В таблицу пишутся только переданные поля, посторонние данные недоступны.
- **Двойные записи.** Каждая сдача = одна строка. Повторная сдача добавит ещё строку — это нормально, видна история попыток.
